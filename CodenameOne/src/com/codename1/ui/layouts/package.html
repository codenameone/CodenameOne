Layout managers allow a {@link com.codename1.ui.Container} to arrange its components by a set of rules that adapt to specific densities (ppi - pixels per inch). A layout manager is an arranging algorithm encapsulated by an abstract class implementation that places components absolutely based on the "hints" received.

Layout Managers that ship with Codename One

{@link FlowLayout} is default layout manager, simple, flexible and with a few caveats.

{@link BorderLayout} is ubiquitous thru Codename One code.

{@link BorderLayout} can also behave differently based on the center behavior flag

{@link BoxLayout} Y axis is a the work-horse of component lists

{@link BoxLayout} X axis is a simpler replacement to flow layout and has grow/no grow variants.


{@link GridLayout} arranges elements in a grid where all elements have an equal size. It can auto adapt the column count.
