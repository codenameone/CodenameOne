![Build a 2D platformer with the Game Builder](/blog/gamebuilder/platformer-6-play.png)

The [Game Builder](/manual/game-builder/) is a visual level editor for the `com.codename1.gaming` API: you lay out a world, set per-object behavior values, preview it live, and **Build** it into your project as a `.game` data file plus a small Java companion that the runtime plays. This is the first of three tutorials — a **2D platformer** ("Coin Run"). Tutorials [2 (board game)](/blog/game-builder-board-game/) and [3 (3D dungeon)](/blog/game-builder-3d-dungeon/) follow.

Every screenshot below is produced by an automated test (`BlogTutorialScreenshots`) that drives the editor through these exact steps, and the same steps are asserted to yield a *valid, runnable* game by `TutorialValidationHarness` — so this isn't aspirational, it builds a working game.

## Getting started

From a **Java 17** Codename One project:

```bash
mvn cn1:create-game-scene -DclassName=com.example.game.CoinRun -Dmode=2d
mvn cn1:gamebuilder
```

The editor opens bound to your project; **Build** writes `src/main/resources/games/<Scene>.game` and (once) the companion `<Scene>.java`. The three concepts you'll use throughout: **Layers** (tile grids vs. freely-placed entity layers), **Assets** (pick a pack, click an asset, click/drag onto the stage), and the **Inspector** (an object's *Object* transform and *Behavior* metadata).

## Step 1 — New 2D scene

Open the project/scene menu and choose **New scene → 2D Platformer**. You get Background / Terrain / Items / Actors layers ready to use.

![New 2D platformer scene](/blog/gamebuilder/platformer-1-new-scene.png)

## Step 2 — Paint the ground

Select the **Terrain** layer, pick the **Grass** tile in the Asset Library, and paint across the bottom row to lay a floor. Tiles snap into grid cells; drag to paint a run of them.

![Paint a grass floor](/blog/gamebuilder/platformer-2-ground.png)

## Step 3 — Place the player

Select the **Actors** layer, pick **Player**, and click where it should start. In the Inspector → *Behavior*, set `lives = 3` and `jumpHeight = 110` (use **Add property** for any custom field). The Inspector's *Object* section edits this one instance — name, position, layer and per-object **Size**.

![Place the player and set behavior](/blog/gamebuilder/platformer-3-player.png)

## Step 4 — Add coins

Pick **Coin** and stamp a few in the air. The preview's default behavior collects them for score automatically, and each coin's `value` is editable in its Behavior section.

![Stamp some coins](/blog/gamebuilder/platformer-4-coins.png)

## Step 5 — Scene rules

Deselect everything to edit the whole scene. In the Inspector set **Gravity** and the **Background** (Sky). These are level-wide properties saved in the `.game` file.

![Scene-wide gravity and background](/blog/gamebuilder/platformer-5-scene.png)

## Step 6 — Play it

Press **Live** — move with the **arrow keys**, **Up/Space** to jump (gravity and tile collision are simulated, coins add to SCORE). Press **Stop** to return; playing never mutates your level.

![Play the platformer live](/blog/gamebuilder/platformer-6-play.png)

## Where your code goes

Win/lose logic lives in the generated companion's behavior hooks, where each sprite's `getUserData()` is the source `GameElement` — so you read the `lives`/`value` numbers you set in the editor. The level is plain data: re-edit it anytime without touching your code, and your logic is preserved across regeneration.
