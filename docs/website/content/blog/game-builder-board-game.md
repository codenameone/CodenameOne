This is the second [Game Builder](/manual/game-builder/) tutorial — a **board game** ("Checkers Start"). If you're new to the editor, start with [Tutorial 1 — the 2D platformer](/blog/game-builder-2d-platformer/); [Tutorial 3 — a 3D dungeon](/blog/game-builder-3d-dungeon/) follows. Board mode renders your grid through an isometric projection, so the same data becomes a clean isometric board at runtime.

As before, every screenshot is regenerated by the `BlogTutorialScreenshots` test and the steps are asserted runnable by `TutorialValidationHarness`.

## Getting started

From a **Java 17** Codename One project:

```bash
mvn cn1:create-game-scene -DclassName=com.example.game.Checkers -Dmode=board
mvn cn1:gamebuilder
```

## Step 1 — New board scene

Choose **New scene → Board**. You get a **Board** (tile) layer and a **Pieces** (entity) layer, placed through an isometric projection.

![New board scene](/blog/gamebuilder/board-1-new-scene.png)

## Step 2 — Lay the board

Select the **Board** layer, pick the **Square** (board tile), and fill the grid to make the playing surface.

![Lay the board tiles](/blog/gamebuilder/board-2-tiles.png)

## Step 3 — Place pieces

Select the **Pieces** layer, pick the **Token** actor, and stamp pieces on their squares. In the Inspector → *Behavior*, tag each piece with `player = 1` (or `2`) and `cell = a1` so your rules code knows whose piece sits where.

![Place tagged pieces](/blog/gamebuilder/board-3-pieces.png)

## Step 4 — Build

**Build** writes the `.game` file storing the board, the pieces and their properties. Your companion reads `el.getInt("player", 0)` / `el.getString("cell", "")` to drive turn logic.

![The finished board ready to build](/blog/gamebuilder/board-4-build.png)

## Variations

The same flow makes a card game — use the **Card** and **Dice** actors with `suit` / `rank` / `sides` metadata — or any grid game whose rules read per-piece properties. Because a level is plain data, you can ship many `.game` files and load whichever the player picks.
