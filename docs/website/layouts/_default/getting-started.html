From zero to a native build

Create the project.
Run it. Build for iOS.

Initializr generates a Maven project you can open in IntelliJ IDEA, NetBeans, Eclipse, or VS Code. Run the simulator locally, then use the cloud service to build for a device.

Before you start
  • A supported JDK and Maven
  • A Maven-capable IDE
  • A free Codename One account for cloud builds
Check the development environment
01

Generate

Create a Java project with Initializr.

Choose the Java template, enter the group and artifact names, and download the generated Maven project. Unpack it into a normal source directory, not a temporary download folder.

{{ partial "cn1-cta.html" (dict "label" "Open Initializr" "href" "https://start.codenameone.com/" "variant" "secondary" "event" "getting-started-step-1" "external" true) }}
02

Verify locally

Run the project in the simulator.

Open the project root in your IDE and let Maven resolve the modules. Run the simulator action supplied by the generated project, or use the command below from the project root.

mvn verify -P simulator -Dcodename1.platform=javase

Expected result: the application opens in the Codename One simulator. You can switch device skins and inspect the component tree without submitting a cloud build.

03

Reach a device

Submit a native build.

Sign in to the Codename One account from the project tools, then choose a device build. Android builds can start immediately. iOS device builds also need Apple signing assets; the certificate wizard guides that setup.

Configure signingApple certificates and Android keystores Open the build dashboardWatch the build and download the result

No Mac required for the cloud path. The build server runs the native toolchain and returns the device binary.

04

First build complete

Install it, then make one real change.

Download the result from the build dashboard and install it on the target device. Change a label, color, or interaction in the shared project, verify it in the simulator, and submit the next build.

You reached the activation point.

The project runs locally and the native build pipeline is connected.

If a step fails

Keep the error attached to the step that produced it.

Check the Java and Maven versions, rerun with Maven error output, and use mvn cn1:get-help when the project offers it. Nothing is transmitted until you review and send the report.
