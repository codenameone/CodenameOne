# LanguageTool accept list for the developer guide.
#
# Each non-blank, non-comment line is a Python regex matched against the
# exact text LanguageTool flagged (m.context[offsetInContext:
# offsetInContext+errorLength]). The match must consume the full flagged
# span; it is also bracketed by word boundaries so a bare "foo" matches
# the standalone word "foo" but not "foobar".
#
# Use this for proper nouns, product names, class identifiers, file
# extensions, and other technical terms that LanguageTool's English
# dictionary does not recognize.
#
# Conventions:
# - Entries are case-sensitive. Use a character class like [Ii] if both
#   capitalizations should be accepted.
# - When LanguageTool reports an actual typo, fix the source instead of
#   allowlisting it here.

# -----------------------------------------------------------------------------
# Generic code-identifier shapes (overwhelmingly safe: a token that mixes
# upper- and lower-case letters or includes digits is almost certainly a
# class name, method name, variable, or other code-flavored identifier
# that the dictionary will not know.)
# -----------------------------------------------------------------------------

# PascalCase (FooBar, FooBarBaz, FooBars). Stops at simple two-letter
# starts like "Of" so it does not swallow real prose.
[A-Z][a-z]+(?:[A-Z][a-zA-Z0-9]+)+s?

# Class names with a leading acronym followed by mixed case
# (URLImage, NSData, JSObject, HMACSHA, XMLWriter, ...).
[A-Z]{2,}[a-z][A-Za-z0-9]*

# camelCase (fooBar, fooBarBaz).
[a-z]+[A-Z][a-zA-Z0-9]+

# Any identifier-shaped token that contains a digit.
[A-Za-z][A-Za-z0-9]*[0-9][A-Za-z0-9]*

# Hyphenated technical tokens (cn1lib-archetype, mac-os-x-desktop, ...).
[A-Za-z][A-Za-z0-9]*(?:-[A-Za-z0-9]+)+

# Underscored identifiers (foo_bar, FOO_BAR).
[A-Za-z][A-Za-z0-9]*(?:_[A-Za-z0-9]+)+

# All-caps tokens (acronyms: UIID, HMAC, URL, ...). Excludes single-letter
# matches; those are handled by LanguageTool's default behavior.
[A-Z]{2,}s?
